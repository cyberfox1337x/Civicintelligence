# Code Review Skill

## Goal

Produce a focused code review with practical findings.

## Steps

1. Read the relevant files fully.
2. Inspect nearby call sites and related modules.
3. Determine intended behavior.
4. Flag the most meaningful issues only.
5. Order findings by severity and impact.
6. Suggest the smallest practical fixes.

## Output

Use:
- Summary
- Findings
- Optional follow-up checks

## Avoid

- nitpicks without impact
- broad rewrite advice
- speculation not supported by code
